✓Notas para la aprobación de John

· Naranja del roleo: en mockup_mailing_plancompleto.html (TANDA B) el mismo badge de roleo se propuso en naranja ("mismo semantic que REVISAR/atención"). En esta solapa lo mantuve consistente con esa decisión visualmente acá en el mock, pero el sistema real de clases de seguimiento.js (seg-alert--*) hoy solo tiene act/warn/bad/info/conf — no existe una variante naranja para esa columna (sí existe para Control BL, vía seg-bdg--rev, pero es una familia de clases distinta). Implementé la alerta con seg-alert--warn (ámbar) porque sumar una clase CSS nueva es tocar index.html, fuera de mi scope esta tanda. Si preferís el naranja exacto del mock, es 1 línea de CSS nueva (`.seg-alert--roleo`) para una tanda siguiente.
· Filtro Sold-to y opción "roleadas" del select urgencia se inyectan en runtime (createElement), no viven en index.html — decisión explícita del alcance de esta tanda (archivos tocados: solo js/features/seguimiento.js + api/seguimiento.js). Funcionan igual para el usuario; si en algún momento se quiere "bajar" ese markup al HTML estático (p.ej. para que otro dev lo vea sin abrir DevTools), es un cambio de ubicación, no de comportamiento.
· "por vencer" se saca como filtro, pero sigue existiendo como bucket (ssbSlaBucket()) — sigue determinando el color ámbar de la columna ATD→límite y el estado "sin pendientes hoy". Solo dejó de ser un chip/opción de filtro clickeable, tal como pediste.
· Semáforo vs. columna Alertas: son complementarios, no redundantes — el semáforo muestra AVANCE (progreso normal), Alertas muestra EXCEPCIONES (algo que necesita acción o está fuera de lo esperado). Una orden puede estar en 4/5 del semáforo (verde, sano) y tener una alerta de roleo al mismo tiempo (algo cambió después).
· co_config_upsert/toggle con documento: implementado con degradación activa — si la migración de la columna todavía no corrió, crear una regla documento=CO (el caso de hoy) sigue funcionando idéntico a como funciona ahora; pedir explícitamente PE/SEG/COA antes de la migración devuelve un 502 con mensaje claro en vez de romper en silencio. El front de Seguimiento no tiene UI de administración de co_config todavía (es un endpoint sin pantalla propia hoy) — quedó fuera del alcance visual de este mock.
· Accesos directos (ítem 51): los 3 ya estaban completos en el código — no hubo que construir nada, solo verificar. Documentado en la sección 7 para que quede explícito que no es una feature nueva de esta tanda, aunque el inventario original lo marcaba "parcial".
· Este archivo es visual únicamente — no pasó por smoke headless (eso corre sobre index.html real, con python3 -m http.server 8899, después del gate de John sobre este mock).
