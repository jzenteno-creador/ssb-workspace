✓Notas para la aprobación de John

· "Bloqueada" como label genérico (rojo) para duplicado-en-lote y formato inválido, en vez de textos distintos por caso — el detail de abajo (gris chico) es el que explica el motivo real. Esto es una decisión del componente bulk-paste.js: no conoce vocabulario de dominio (es el mismo componente que va a usar Tarifas Terrestres), así que el badge queda genérico y el texto explicativo lo pone cada feature. Si preferís textos de badge más específicos ("Duplicada", "Formato") puedo agregar esa granularidad sin romper el contrato.
· "Ya generado para esta orden" es warning, no error — pedido explícito del GO: no bloquea, se re-genera el PDF (el upsert on_conflict=orden,certificado_numero ya soporta este caso hoy). Si un certificado se pegó dos veces EN EL MISMO LOTE, ese sí bloquea (fila 3 del ejemplo) — evita 2 llamadas al mismo endpoint en el mismo click.
· Progreso secuencial, nunca paralelo — el botón muestra "Cargando i/N…" fila a fila. Con lotes grandes (100+ filas con ZIPs reales) esto puede tardar varios minutos (~6-10s por fila medido vía el gateway n8n de Drive) — dejé el texto del botón como único indicador de avance, sin barra de progreso animada (menos JS, mismo dato).
· Buscador con limit(100), harcodeado — no hay paginación. Si en la práctica una búsqueda por texto corto (ej. "AR004") devuelve más de 100 filas, se recorta silenciosamente en orden de updated_at desc (las más recientes primero). Avisame si preferís un mensaje "hay más resultados, afiná la búsqueda" cuando pega el tope.
· Reasignar NO toca pdf_nombre/pdf_drive_url/zip_drive_url — solo cambia la columna orden. El PDF físico en Drive sigue llamándose {orden_vieja}_{cert}_CO.pdf hasta que alguien lo regenera (por eso el ofrecimiento inmediato de "Generar ahora" al final del flujo) — el warning en la respuesta lo deja explícito.
· Sin columna de auditoría dedicada para "quién reasignó" — la tabla certificados_origen no tiene updated_by (verificado en vivo, solo generado_por). No lo pisé para no perder el dato de quién generó el PDF originalmente; el actor de la reasignación queda solo en la RESPUESTA del endpoint (no persistido). Si querés trazabilidad persistida, es una migración chica (columna reasignado_por + reasignado_at) — quedó fuera de esta tanda a propósito.
· Este archivo es visual únicamente — el pegado masivo real corre contra js/shared/bulk-paste.js (componente reusable, cero CSS nuevo, solo badge global + inline styles) y /api/certificado-origen con action:'reasignar' nuevo; no pasó por navegador real todavía (eso es el smoke headless del gate, separado de este mock).
